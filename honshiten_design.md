# 本支店会計 仕訳ドリル 設計書（honshiten.html 用）

cash.html（現金預金）と同形式のドリルとして実装する。この文書は問題・解答・解説のセット（コンテンツ）と UI 上の要件を定義する。実装時は cash.html の構造（CONCEPTS 配列・チップ UI・grade 関数・selfTest）を流用すること。

## 1. UI 方針

- 基本は cash.html と同じ「科目チップを 借方 → 貸方 の順にタップ」形式。金額の入力は求めない（金額は問題文と解説に表示するのみ）。
- **記帳側（本店側／支店側）の選択を回答の一部にする。**
  - カテゴリ「未達取引」：問題文は取引と未達の事実だけを述べ、**どちらの帳簿で仕訳するかを学習者に選ばせる**（「本店側」「支店側」の 2 ボタン。cash.html の銀行残高±セレクタと同じ仕組みで、こちらは常時表示・回答必須）。未達整理は「まだ記帳していない側で仕訳する」ことが論点の核心のため。
  - カテゴリ「基本取引」：両者が仕訳を行う取引なので、問題文側で「このとき**本店側**（または**支店側**）の仕訳を示しなさい」とランダムに指定する。セレクタは不要（または指定側を自動選択状態で表示）。
  - カテゴリ「支店間取引」：問題文で計算制度（支店分散／本店集中）と記帳側（大阪支店・神戸支店・本店）を指定する。**支店分散計算制度の本店は「仕訳なし」**が正解になるので、仕訳なしボタンを活かせる。
  - カテゴリ「決算整理」：問題文で記帳側を明示する。
- カテゴリ色分け（cash.html の cat-〇〇 と同様）：基本取引／未達取引／支店間取引／決算整理 の 4 色。

### 科目チップ（ACCOUNTS）と略称（ABBREV）案

`現金`(げ) `当座預金`(と) `売掛金`(うK) `買掛金`(KK) `仕入`(し) `売上`(う) `営業費`(え) `本店`(本) `支店`(支) `本店より仕入`(本仕) `支店へ売上`(支う) `大阪支店`(大阪) `神戸支店`(神戸) `損益`(損) `総合損益`(総損) `繰延内部利益`(繰内) `繰延内部利益控除`(控除) `繰延内部利益戻入`(戻入)

チップが多いので、出題中のカテゴリに応じて不要なチップを薄く（または非表示に）してよい。ただし「本店／支店」「仕入／本店より仕入」の混同を試す意味で、基本・未達カテゴリでは両方見せること。

## 2. 金額設計（割り切れルール）

金額は採点対象ではないが、問題文のリアリティと解説の計算例のために付す。CLAUDE.md の方針どおり、必ず整数になる基数設計とし、node で大量生成テストしてから出すこと。

- 送金・立替・回収額：`10,000 × rand(5〜30)` 円
- 商品（原価送付）：原価単価 `1,000 × rand(2〜8)` 円 × 数量 `10 × rand(1〜5)` 個
- 商品（振替価格送付）：原価に **20％の利益を加算**（×1.2）。原価単価が 1,000 の倍数なので必ず整数
- 内部利益の計算問題：先に内部利益 `X = 1,000 × rand(3〜12)` を決め、**振替価格ベースの商品額を 6X** とする（利益率 20％ ⇒ 内部利益 = 6X × 0.2/1.2 = X で必ず整数）

## 3. 論点一覧（CONCEPTS）

記法：`本店側 (借)〇〇 (貸)〇〇 ／ 支店側 (借)〇〇 (貸)〇〇`。wordings は言い回しのバリエーション（実装時は 2〜3 個ずつ用意。《金額》等は生成値を埋め込む）。

### カテゴリ A：基本取引（問題文で記帳側を指定。1 論点から本店側／支店側の 2 問が生成できる）

| id | 取引 | 本店側 | 支店側 |
|---|---|---|---|
| A1 | 本店が支店に現金《A》円を送付した | 支店／現金 | 現金／本店 |
| A2 | 支店が本店に現金《A》円を送金した | 現金／支店 | 本店／現金 |
| A3 | 本店が支店に商品（原価《A》円）を**原価のまま**発送した | 支店／仕入 | 仕入／本店 |
| A4 | 本店が支店に商品（原価《A》円）を**原価に20％の利益を加算した価格**で発送した | 支店／支店へ売上 | 本店より仕入／本店 |
| A5 | 支店が A4 で受け取った商品の一部（振替価格《B》円）を本店に**戻した**（返品） | 支店へ売上／支店 | 本店／本店より仕入 |
| A6 | 支店が本店の売掛金《A》円を現金で回収した | 支店／売掛金 | 現金／本店 |
| A7 | 本店が支店の売掛金《A》円を現金で回収した | 現金／支店 | 本店／売掛金 |
| A8 | 本店が支店の買掛金《A》円を現金で支払った | 支店／現金 | 買掛金／本店 |
| A9 | 支店が本店の買掛金《A》円を現金で支払った | 買掛金／支店 | 本店／現金 |
| A10 | 本店が支店の営業費《A》円を現金で立替払いした | 支店／現金 | 営業費／本店 |
| A11 | 支店が本店の営業費《A》円を現金で立替払いした | 営業費／支店 | 本店／現金 |

wordings 例（A4）：
- 「本店は支店に対し、原価《A》円の商品を、原価に20％の利益を加算した価格で発送した。**本店側**の仕訳を示しなさい。」
- 「本店から支店へ、商品（原価《A》円）が振替価格（原価の20％増し）で送付された。**支店側**の仕訳を示しなさい。」

explain 例：
- A1〜A2『本店勘定と支店勘定は**照合勘定**であり、互いに貸借逆で残高が一致する。送付側は相手勘定の借方（債権の増加）、受取側は相手勘定の貸方（債務の増加）を記帳する。』
- A3『原価による発送では内部利益が生じないため、本店は「仕入」を貸方に立てて原価を支店へ振り替える。支店は「仕入」の借方で受け入れる。』
- A4『振替価格による発送では、本店は「支店へ売上」、支店は「本店より仕入」という内部取引専用の勘定を用いる。両勘定は決算で相殺され、含まれる内部利益は繰延処理の対象となる。』
- A6〜A11『相手のために立替・回収をしたときは、自分の帳簿では相手勘定（本店／支店）で処理し、相手の帳簿では自分に対する勘定と本来の科目（売掛金・買掛金・営業費）で処理する。**営業費はあくまで費用の帰属先（本店の費用か支店の費用か）で計上する側が決まる**点に注意。』

wordings 例（B4。写真の資料の言い回しに寄せる）：
- 「支店は、本店の売掛金《A》円を本店に代わって小切手で回収したが、本店に未達である。」
- 「決算日現在、支店が行った本店の売掛金《A》円の回収の通知が、本店に到達していなかった。」

explain 共通の型：『未達取引は、**通知（または現物）がまだ届いていない側**の帳簿にだけ記帳が欠けている。したがって未達である側（＝問題文で「〇〇に未達」とされた側）で、届いたものとして仕訳を行う。相手側の仕訳はすでに済んでいるので不要。』
- B1 追加『商品の未達は支店側で「本店より仕入」を計上する。未達商品は支店の期末商品棚卸高にも含める（内部利益控除の対象になる）。』
- B2〜B3 追加『現金の未達は受け取る側で現金（送金内容によっては当座預金）を計上し、相手勘定を貸方に立てる。』

### カテゴリ C：支店間取引（問題文で計算制度と記帳側を指定）

取引例：「大阪支店は神戸支店に現金《A》円を送付した」／「大阪支店は神戸支店に商品（原価《A》円）を原価で発送した」

| 制度 | 大阪支店（発送側） | 神戸支店（受取側） | 本店 |
|---|---|---|---|
| 支店分散計算制度・現金 | 神戸支店／現金 | 現金／大阪支店 | **仕訳なし** |
| 支店分散計算制度・商品 | 神戸支店／仕入 | 仕入／大阪支店 | **仕訳なし** |
| 本店集中計算制度・現金 | 本店／現金 | 現金／本店 | 神戸支店／大阪支店 |
| 本店集中計算制度・商品 | 本店／仕入 | 仕入／本店 | 神戸支店／大阪支店 |

1 取引 × 2 制度 × 3 記帳側 ＝ 実質 12 パターンが生成できる。

explain：
- 分散『支店分散計算制度では、支店同士が相手支店勘定を直接設けて記帳する。**本店は取引に関与しないため仕訳なし**。』
- 集中『本店集中計算制度では、各支店は本店を相手に取引したとみなして「本店」勘定で記帳し、**本店が両支店の勘定を付け替える仕訳**（借方：受取側支店、貸方：発送側支店）を行う。』

### カテゴリ D：決算整理（問題文で記帳側を明示。本店の帳簿で総合損益・内部利益を扱う前提）

| id | 問題文の骨子 | 正解 |
|---|---|---|
| D1 | 支店の期末商品のうち本店からの仕入分は《6X》円である（本店は原価に20％の利益を加算して発送）。内部利益を控除する（本店側） | 繰延内部利益控除／繰延内部利益 |
| D2 | 支店の期首商品に含まれていた内部利益は《X》円である。決算につき戻し入れる（本店側） | 繰延内部利益／繰延内部利益戻入 |
| D3 | 支店は当期純利益《A》円を計上した。支店側の損益振替の仕訳 | 損益／本店 |
| D4 | 支店の当期純利益《A》円の報告を受けた。本店側の仕訳（総合損益勘定を設けている） | 支店／総合損益 |
| D5 | 支店は当期純損失《A》円を計上した。支店側の仕訳 | 本店／損益 |
| D6 | 支店の当期純損失《A》円の報告を受けた。本店側の仕訳（総合損益勘定を設けている） | 総合損益／支店 |

explain：
- D1『期末商品に含まれる内部利益 = 振替価格ベースの金額 × 0.2/1.2 =《X》円。当期の利益から控除するとともに「繰延内部利益」として次期に繰り延べる。』
- D2『期首商品の内部利益は前期に控除済みなので、当期は戻し入れて当期の利益に加算する。』
- D3〜D6『支店の純損益は支店の損益勘定から「本店」勘定へ振り替え、本店はこれを「支店」勘定で受けて総合損益勘定に集計する（総合損益勘定を設けない場合は本店の損益勘定）。純損失なら貸借が逆になる。』

### 発展（今回は実装対象外。将来の拡張候補としてメモ）
